## 答案第一个方法是通过堆来排序，同样用哈希表记录频次，可以通过这个记录频次
`occurrences.put(num, occurrences.getOrDefault(num, 0) + 1);`
### 这个方法用堆(也就是一个优先队列，它会按某种优先级自动排序)怎么初始化这样一个队列呢
`PriorQueue<int[]> queue = new PriorQueue<int[]>(new Comparator<int[]>())` 
### 这个comparator就是告诉queuq怎么优先排序的。通常会在下一行写一个compare函数，compare(a, b) < 0 表示：a 应该排在 b 前面。
`compare(a, b) > 0`
### 表示：a 应该排在 b 后面。这个是规则，按规则编就可以了，通常a-b的话就是小堆序，那么怎么用这个堆来找出频率的前k位呢，就是遍历哈希表，如果堆的大小是等于k的，那么比较遍历数对频率大小和堆顶频率最小的那个，假如大过堆顶的，把堆顶的踢了，把这个插进去，用queue.peek来选择，queue.poll来踢走，queue.offer插入
## 答案第二个方法是快速排序，把哈希表里的数对全部添加到一个List<int[]>里面，对这个表快速排序，
### 快速排序算法应该学习
### `int picked=(int)(Math.random()*(end-start+1)+start)`随机选择一个位置
`Collections.swap(values,picked,start)`在表value里换picked和start位置，pivot=value.get(start)[1]把它出现次数做基准，
`index=start，`for i从start+1开始到end，出现次数小于pivot的调换到左边，`Collections.swap(values,index+1,i);index++,
然后判断前k大在哪边，`if(k<=index-start),qsort(values,start,index-1,ret,retIndex,k)`,否则k比左边数字还大，左边的也一定属于答案，把它们都放进ret里ret[retIndex++] = values.get(i)[0];然后再对后面的快排，qsort(valus,index+1,end,ret,retIndex,k-(index-start+1))
